# POM generator — scaffold page objects

The CLI at `scripts/generate-pom.ts` generates Page Object Model (POM) TypeScript from:

- **Browser** — live DOM scan via Playwright  
- **Mobile** — Appium page-source XML  
- **Desktop** — Accessibility tree (live app) or a JSON spec  
- **API** — endpoint JSON spec  

**Desktop and mobile paths are unchanged** from earlier releases; only the **browser** pipeline gained optional libraries and flags.

---

## 1. Invocation

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts <platform> [options]
```

`npm run pom:gen` runs the same script if defined in `package.json`.

---

## 2. Browser

### Default mode (IDriver-friendly POM)

Emits a class extending **`DriverPage`** with `this.element(selector)` fields. Scanning uses **`DOMScanner`** plus **`SelectorStrategy`** in `src/drivers/browser/pom/` (interactive nodes, visibility, open shadow roots, semantic regions, stable selector ordering).

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts browser \
  --url https://www.example.com \
  --class-name ExamplePage \
  --update-index
```

| Flag | Required | Default | Description |
|------|----------|---------|-------------|
| `--url` | yes | — | URL to open (headless Chromium) |
| `--class-name` | no | Host-derived + `Page` | Exported class name |
| `--out` | no | `tests/pom/browser/<kebab>.ts` | Output path |
| `--static-prop` | no | `entryUrl` | Static URL property name |
| `--max-elements` | no | 80 | Max fields (capped at 500) |
| `--update-index` | no | false | Append `export` to `tests/pom/index.ts` |

### Enhanced mode (Playwright `Page` / `PageObject`)

Produces a POM extending **`PageObject`** with `this.locator(...)`, region-grouped properties, and an optional `fillForm` helper when multiple form controls are detected. Uses the same scan output but **`POMGenerator`** for codegen and atomic writes.

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts browser \
  --url https://www.example.com \
  --enhanced \
  --base-class PageObject \
  --update-index
```

| Flag | Required | Default | Description |
|------|----------|---------|-------------|
| `--enhanced` | no | false | Use `POMGenerator` + region grouping + helpers |
| `--base-class` | no | — | Only with `--enhanced`: `PageObject` or `DriverPage` |

Without `--enhanced`, the CLI always calls **`generateBrowserPom`** → **`DriverPage`**. **`--base-class` does not apply** in that path.

### Selector priority (browser)

Implemented in **`src/drivers/browser/pom/selector-strategy.ts`**:

1. `data-testid`  
2. Stable `id` (framework-generated ids are skipped — see `isAutoGeneratedId`)  
3. `aria-label`  
4. `name`  
5. Role + visible text (heuristic)  
6. `placeholder`  
7. Text / href / XPath fallbacks  

To tune heuristics, edit **`selector-strategy.ts`** or **`dom-scanner.ts`**, not ad hoc logic in the CLI (except orchestration).

### Reusing the library in code

You can scan or generate from any test or script that holds a Playwright **`Page`**:

```typescript
import { DOMScanner } from '../src/drivers/browser/pom/dom-scanner';
import { POMGenerator } from '../src/drivers/browser/pom/pom-generator';

const scan = await new DOMScanner({ maxElements: 120 }).scan(page);
const { outputPath } = new POMGenerator().generateAndWrite(scan, {
  className: 'MyPage',
  outputPath: 'tests/pom/browser/my-page.ts',
  baseClass: 'PageObject',
  updateBarrel: true,
});
```

## 3. Mobile

Parse Appium XML (`XCUIElementType*` / `android.widget.*`).

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts mobile \
  --source ./page-dump.xml \
  --class-name LoginMobileScreen
```

| Flag | Required | Default |
|------|----------|---------|
| `--source` | yes | — |
| `--class-name` | no | `GeneratedMobileScreen` |
| `--out` | no | `tests/pom/mobile/<kebab>.ts` |
| `--update-index` | no | false |

## 4. Desktop (macOS / Windows)

### Live Accessibility scan

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts desktop \
  --app Notes \
  --class-name NotesScreen \
  --max-elements 100
```

Uses **`MacOSAdapter`** or **`WindowsAdapter`** depending on `process.platform`. The app must be running; Accessibility must be allowed.

### JSON spec

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts desktop \
  --json ./notes-elements.json \
  --class-name NotesScreen
```

```json
{
  "elements": [
    { "property": "newNote", "selector": "New Note" },
    { "selector": "Search" }
  ]
}
```

| Flag | Required | Notes |
|------|----------|--------|
| `--app` or `--json` | one required | Live scan vs static list |
| `--class-name` | no | Default: `<AppName>Screen` or `GeneratedDesktopScreen` |
| `--max-elements` | no | Cap when using `--app` |
| `--out`, `--update-index` | no | Same as browser |

## 5. API

```bash
npx ts-node --project scripts/tsconfig.json scripts/generate-pom.ts api \
  --json ./user-api.json
```

```json
{
  "className": "UserApi",
  "comment": "User management endpoints",
  "endpoints": [
    { "name": "listUsers", "method": "get", "path": "/users" },
    { "name": "getUser", "method": "get", "path": "/users/:id" },
    { "name": "createUser", "method": "post", "path": "/users" },
    { "name": "updateUser", "method": "put", "path": "/users/:id" },
    { "name": "deleteUser", "method": "delete", "path": "/users/:id" }
  ]
}
```

Path segments like `:id` become method parameters.

## 6. Generated shapes (reference)

### Browser — `DriverPage` (default CLI)

```typescript
import { DriverPage } from '../../../src/pom/driver-page';

/** Auto-generated (browser) — https://www.example.com */
export class ExamplePage extends DriverPage {
  static readonly entryUrl = 'https://www.example.com';
  readonly signIn = this.element('a:has-text("Sign In")');

  async open(): Promise<void> {
    await this.navigate(ExamplePage.entryUrl);
  }
}
```

### Browser — `PageObject` (`--enhanced --base-class PageObject`)

```typescript
import { PageObject } from '../../../src/drivers/browser/pom/page-object';
import { Page } from 'playwright';

/** Auto-generated POM for https://www.example.com */
export class ExamplePage extends PageObject {
  static readonly entryUrl = 'https://www.example.com';
  readonly signIn = this.locator('a:has-text("Sign In")');

  async open(): Promise<void> {
    await this.navigate(ExamplePage.entryUrl);
  }
}
```

### Desktop

```typescript
import { DesktopPage } from '../../../src/drivers/desktop/pom/desktop-page';
import { DesktopDriver } from '../../../src/drivers/desktop/desktop-driver';

export class NotesScreen extends DesktopPage {
  readonly newNote = this.element('New Note');

  constructor(driver: DesktopDriver) {
    super(driver);
  }
}
```

### Mobile — see `tests/pom/mobile/` samples; pattern is `MobileScreen` + `MobileDriver` constructor.

### API — extends `EndpointGroup`; see existing `tests/pom/api/` examples.
